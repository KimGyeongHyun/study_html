1. state 변수 하나만으로 모달창 컨트롤하기

    기본 동작 : 메인 페이지에서 버튼을 눌러 모달창을 띄움,
    모달창에서 X 표시를 눌러 모달창 닫기
    
    메인 페이지의 버튼, 모달창의 X 에서 state 변수를 바꿔야 하는데
    모달창은 src 폴더 안 modal.jsx 에 구현되어 있으므로
    state 변수를 modal.jsx 에 전달할 필요가 있었다

    state 변수는 함수를 통해 값을 갱신하므로 변수를 전달하는 대신
    state 변수를 갱신하는 함수를 props 로 전달했다
    모달창에서 X 표시를 눌렀을 때 해당 함수가 실행되면서
    자연스럽게 App.js 의 setState 함수에 접근할 수 있게 된다

2. node_modules 을 사용하기 위해서 import 할 때 src 외부를 import 할 수 없는 문제

    src 외부 디렉토리를 참조할 수 없다는 경고창이 뜸
    
    import 할 때 기본적으로 node_modules 파일로 경로로 들어가기 때문에
    상대경로를 임의로 적지 않아도 된다

3. bootstrap 의 컴포넌트를 사용할 때 임의로 클래스를 넣을 때
    적용되지 않는 현상

    bootstrap 등의 프레임워크를 사용하면 
    내가 만든 클래스들이 오작동할 수 있다는 것을 인지해야 한다
    (bootstrap radio button 이 모달창보다 위에 있음,
    background-color 가 적용되지 않음)

    bootstrap 컴포넌트를 사용할 땐 구현되어 있는 내부 속성을 우선적으로
    사용해야 한다
    이후 세부적인 설정을 하면 된다 (컴포넌트 크기 등)
