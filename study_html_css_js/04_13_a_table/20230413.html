다이나믹 프로그래밍 수행

지금 작동방식은 str 출력 메소드로부터 계산 메소드가 불리는 형식
다이나믹 프로그래밍을 적용한다면 :
str 출력 메소드로부터 계산 메소드를 부르는 것보단, 계산 메소드를 모두 수행한 뒤 계산 결과를 저장하고 따로 출력하는 것이 바람직함

클래스마다 하는 역할 하나만 하도록 수정 필요

GameInfo 클래스는 제일 큰 클래스이며 모든 클래스를 인스턴스로 가짐
UnitCalculator 와 PlayerCalculator 클래스는 각각의 역할에 따라 계산하고 정보를 보여줌
하지만 위의 두 클래스는 각각의 역할에 따른 정보에 한해서만 계산할 수 있기 때문에 두 정보를 모두 필요로하는 계산의 경우 두 클래스 모두 접근 불가능함
따라서 현재 GameInfo 클래스 안에 해당 계산을 수행하는 메소드가 존재하며, 이는 Single responsibility principle 에 어긋남
