기존 문제
기능에 문제는 없지만 body 를 감싸는 div 가 
class="~~" class="scrollspy" 의 형식으로
클래스가 두 번씩 쓰여 있었음
(html 검사 결과 에러 발생)

class="~~" 를 제거할 경우 전체 스크롤이 움직이는 것이 아닌
scrollspy div 의 스크롤이 움직이는 것이기 때문에
AOS 와 video 스크롤 이벤트가 발생하지 않음

보아하니, 부트스트랩에서 scrollspy 를 다루는 방식은
scrollspy 를 가지는 요소의 스크롤을
nav 에서 컨트롤 하는 것

그에 따라 scrollspy 의 스크롤이 변할 뿐
window 의 스크롤이 변하지 않기 때문에
스크롤 이벤트를 수행할 수 없음

이전의 class="~~" 오류 코드가 해당 scrollspy 요소를
자연스럽게 window 스크롤로 트래킹하는 역할을 해줬던 것

궁극적으로 body 를 scrollspy 로 지정하면 문제는 해결되지만
부트스트랩에서 지원하지 않음

따라서 해당 문제를 해결할 수 있는 방법들은

1. 윈도우 스크롤 이벤트를 어떻게든 가져온다 
    -> 그대로 오류 코드를 쓰거나, 해결 불가능
    부트스트랩에서 지원해 주지 않음
2. scrollspy 의 스크롤 이벤트를 가져와 사용한다
    -> AOS 를 직접 만져야 할 수도 있음
3. nav 를 window 로 트래킹하도록 js 를 직접 구성한다
    -> 제일 명확하지만 비효율적

2 번의 해결 방법이 제일 좋아 보임
AOS 를 직접 만지는 방법은 라이브러리 의도상 맞지 않아보임
따라서 AOS 를 직접 js 로 다시 구상하여 만들어야 할 듯 함
물론 video js 도 다시 구성해야 함 (완료)

nav.css 에 scrollspy-example 클래스가 height: 100vh 로 있었음
의도치 않은 동작을 한 것이 이것 때문이었다는 것을 코드 정리하면서 알았음

모바일 크롬 환경에서 상단바가 div 를 차지하고, 없어지지 않음

위로 드래그하여 아래 컨텐츠를 볼 때 상단바가 없어졌었지만
지금은 계속 남아있음
또한, 상단바가 header 의 윗부분을 먹어버려 위쪽이 보이지 않음

body 의 스크롤 바가 아니라 div 스크롤이기 때문이라고 판단

윈도우의 스크롤이 바뀔 일이 없기 때문에
상단바가 그대로 존재하는 것

윈도우 스크롤 변화 없이 큰 div 를 만들어 스크롤을 붙이면
위와 같은 현상이 발생함

해결 방법
1. 큰 div 를 사용하지 않는다

다시 scrollspy 가 body 를 가져가게 해야 한다
코드를 정리하면서 scrollspy-example 클래스의 height: 100vh 스타일 속성을
뒤늦게 확인했기 때문에
다시 body 로 뺐을 때 되지 않을까 하는 기대감이 있음

하지만 역시 scrollspy 를 body 가 가져갈 경우 nav 가 트래킹하지 못 하는
현상이 발생했다

결국 남아있는 방법은 scrollspy 를 직접 구성하는 방법과,
이전 오류코드를 사용하는 방법 2가지 뿐이다

둘 다 좋지 않은 방법이라고 생각하며 다른 방법이 있는지 검색해봐야 할 듯
