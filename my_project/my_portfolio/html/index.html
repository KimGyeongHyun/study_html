Mind set

Logic

코드를 구성할 때 항상 효율적으로 구성하려는 습관을 가지고 있습니다.

최대한 간단하게, 필요한 로직만 사용하고, 다른 사람이 보기에 단순하고 깔끔하게 구성하려고 노력합니다.

동일한 기능을 수행하는 컴포넌트가 여러개 있다면 코드를 두 번 작성하는 대신 함수를 작성하여 컴포넌트에 사용합니다.

Modularize

코드를 구성할 때 모듈화를 하지 않으면 다른 컴포넌트로간의 의도치 않은 동작을 할 수 있고, 정리되지 않은 코드로 인한 가독성 저하, 그에 따른 코드 수정이 어렵다고 생각합니다.

특정 작업만을 수행하는 코드를 개별로 구성하면 해당 작업의 오류 코드를 쉽게 찾아 수정하기 편하고, 다른 요소 간의 간섭을 덜 걱정할 수 있습니다.

건설로 비유하자면 모듈화는 건설 이전의 설계된 도면이라고 생각합니다. 전반적인 설계 없이 건물을 구성하면 금방 무너져 내릴 것입니다.

Open Mind

코드를 혼자 작성하면 해당 언어가 금방 익숙해진다는 장점이 있습니다. 하지만 주변 도움 없이 혼자 코드를 작성하면 손쉽게 해결할 수 있는 문제점에 부딪혀 혼자 고민하게 됩니다.

또한 외부에서 코드를 참고할 때 다양하게 찾아보지 않는다면 좋은 코드를 찾지 못할 수 있습니다.

복잡한 기능이 구현된 유용한 외부 라이브러리들을 활용하면 기능을 직접 구현하는 시간을 줄일 수 있습니다. 필요한 기능이 있다면 해당 기능만 참고하여 사용할 수 있습니다.
